# Task: AuthService and Auth Types

## Goal
Build the core of the `keycloak-auth` feature: an injectable `AuthService` with full session management, plus TypeScript types for auth state. This replaces the React `AuthProvider` context + `useAuth` hook + `auth.api.ts` with a singleton Angular service.

## Context
The existing React implementation: `projects/application/frontend/app/src/features/keycloak-auth/services/auth.api.ts` (API calls), `hooks/use-auth.tsx` (state), `AuthProvider.tsx` (context). The Angular version is a single `AuthService` that owns all auth state as RxJS Subjects.

Backend auth endpoints (unchanged):
- `POST /auth/login` ‚Äî body: `{ username, password }` ‚Äî sets HTTP-only cookie
- `POST /auth/logout` ‚Äî clears cookie
- `GET /auth/check` ‚Äî returns `{ authenticated: boolean, user?: AuthUser, theme?: 'dark' | 'light' }`
- `POST /auth/refresh` ‚Äî refreshes the JWT cookie; returns `{ success: boolean }`

The inactivity timeout (30 min) and proactive token refresh (every 4 min) are managed here. The proactive refresh uses `setInterval`; inactivity tracking is via an Angular `APP_INITIALIZER` or activity interceptor that resets a timer.

## What to Build

Create `src/features/keycloak-auth/` with:

- `types.ts` ‚Äî TypeScript interfaces:
  - `AuthUser { id: string; username: string; email: string; roles: string[]; }`
  - `AuthState { authenticated: boolean; user: AuthUser | null; loading: boolean; }`
  - `Permission` (matches existing permission system in React app ‚Äî see `permissions/` directory)
- `auth.service.ts` ‚Äî `@Injectable({ providedIn: 'root' })`:
  - `private http = inject(HttpClient)`
  - `authState$: BehaviorSubject<AuthState>` ‚Äî current auth state
  - `isAuthenticated$: Observable<boolean>` ‚Äî derived from authState$
  - `currentUser$: Observable<AuthUser | null>` ‚Äî derived from authState$
  - `login(username: string, password: string): Observable<void>` ‚Äî calls `POST /auth/login`, then `checkAuth()`
  - `logout(): Observable<void>` ‚Äî calls `POST /auth/logout`, resets state
  - `checkAuth(): Observable<AuthState>` ‚Äî calls `GET /auth/check`, updates state; returns resolved state
  - `refresh(): Observable<boolean>` ‚Äî calls `POST /auth/refresh`
  - `startProactiveRefresh(): void` ‚Äî sets up `setInterval` for every 4 minutes
  - `stopProactiveRefresh(): void`
  - `hasPermission(permission: string): boolean` ‚Äî synchronous check against current user roles (same logic as `use-permission.tsx`)
- `permissions/` ‚Äî port the permission derivation logic from React:
  - `permissions.ts` ‚Äî `hasPermission(user: AuthUser | null, permission: string): boolean` pure function; matches existing `permissions/` logic

- `index.ts` ‚Äî barrel: `export { AuthService } from './auth.service'; export * from './types'; export { hasPermission } from './permissions/permissions';`
