# Task: App Layout Component and Layout Service

## Goal
Build the `layouts` feature: the `AppLayoutComponent` shell that wraps all authenticated pages with the header + sidenav + content area, and the `LayoutService` that manages responsive sidebar state. Replaces the React `layouts/` feature.

## Context
The existing React layout: `projects/application/frontend/app/src/features/layouts/app-layout.tsx` and `layout-context.tsx`. It wraps authenticated routes with a header + sidebar + main content area, and responds to viewport breakpoints.

The Angular version uses Angular Material's `<mat-sidenav-container>` + `<mat-sidenav>` + `<mat-sidenav-content>`. The `LayoutService` uses Angular CDK's `BreakpointObserver` to detect mobile vs. desktop and switch the sidenav between `mode="over"` (mobile/temporary) and `mode="side"` (desktop/permanent).

`AppHeaderComponent` (task t-89679546b27d) and `NavigationSidebarComponent` / `NavigationDrawerComponent` (task t-b3fdb1c0f543) are composed here.

## What to Build

Create `src/features/layouts/` with:

- `layout.service.ts` — `@Injectable({ providedIn: 'root' })`:
  - `private breakpoints = inject(BreakpointObserver)` from `@angular/cdk/layout`
  - `isMobile$: Observable<boolean>` — true when viewport matches `Breakpoints.Handset`
  - `sidenavOpen$: BehaviorSubject<boolean>` — controls drawer open state
  - `toggleSidenav(): void`
  - `openSidenav(): void`, `closeSidenav(): void`
- `app-layout.component.ts` — standalone `@Component`:
  - `selector: 'app-layout'`
  - Imports: `[MatSidenavModule, AppHeaderComponent, NavigationSidebarComponent, NavigationDrawerComponent, RouterModule, CommonModule, AsyncPipe]`
  - Injects `LayoutService`
  - Template: `<mat-sidenav-container>` with conditional sidenav based on `isMobile$`; `<mat-sidenav-content>` contains `<app-header>` + `<router-outlet>`
  - On mobile: `<mat-sidenav mode="over">` with `NavigationDrawerComponent`
  - On desktop: `<mat-sidenav mode="side" opened>` with `NavigationSidebarComponent`
  - Hamburger click from `AppHeaderComponent` calls `LayoutService.toggleSidenav()`
- `app-layout.component.html` — template
- `index.ts` — barrel: `export { AppLayoutComponent } from './app-layout.component'; export { LayoutService } from './layout.service';`

## Acceptance Criteria
- [ ] `AppLayoutComponent` renders header + sidenav + `<router-outlet>`
- [ ] Desktop: sidenav is permanent (`mode="side"`)
- [ ] Mobile: sidenav is temporary (`mode="over"`) and toggled by hamburger
- [ ] `LayoutService.isMobile$` uses `BreakpointObserver`
- [ ] `AppLayoutComponent` is standalone
- [ ] `ng build` succeeds
- [ ] Type-check passes
