# Task: Chat Services (ChatService + MessageListService)

## Goal
Build the services layer of the `mastra-agents` feature: `ChatService` (manages active chat state, sends messages, handles streaming via Socket.IO) and `MessageListService` (manages the conversation list / chat history sidebar). These replace the React `ChatProvider` context and `chat-context.tsx`.

## Context
The existing React implementation: `projects/application/frontend/app/src/features/mastra-agents/chat/ChatProvider.tsx` (chat state + socket), `chat/chat.service.ts` (Socket.IO calls), `chat-history/` (sidebar state). The Angular version converts these to injectable services using RxJS.

The `WebSocketService` (task t-557fd45ea80e) is injected for Socket.IO. The `AuthService` (task t-5a23bd61342d) provides the current user/agent context.

Backend WebSocket:
- Namespace: `/mastra-chat`
- Emit `chat-message`: `{ message: string; agentId: string; threadId?: string }`
- Receive `chat-chunk`: `{ content: string; threadId: string }` (streaming)
- Receive `chat-complete`: `{ threadId: string }`
- Receive `chat-error`: `{ error: string }`
- Namespace: `/mastra-chat-history` for conversation list

## What to Build

Create/extend `src/features/mastra-agents/` with:

- `services/chat.service.ts` — `@Injectable({ providedIn: 'root' })`:
  - `private ws = inject(WebSocketService)`
  - `messages$: BehaviorSubject<ChatMessage[]>` — current conversation messages
  - `isStreaming$: BehaviorSubject<boolean>` — true while receiving chunks
  - `currentThreadId$: BehaviorSubject<string | null>`
  - `sendMessage(text: string, agentId: string): void` — emits `chat-message`, subscribes to chunks
  - Handles `chat-chunk` by appending content to the last message (or building a streaming message)
  - Handles `chat-complete` by marking `isStreaming$` false
  - Handles `chat-error` by adding error message
  - `clearMessages(): void`
  - `initialize(): void` — connects the WebSocket namespaces
- `services/message-list.service.ts` — `@Injectable({ providedIn: 'root' })`:
  - Manages conversation thread list from `/mastra-chat-history` namespace
  - `conversations$: BehaviorSubject<Conversation[]>`
  - `loadConversations(): void`
  - `selectConversation(threadId: string): void` — updates `ChatService.currentThreadId$` and loads messages
- `types.ts`:
  - `ChatMessage { id: string; role: 'user' | 'assistant'; content: string; timestamp: Date; isStreaming?: boolean; }`
  - `Conversation { threadId: string; title: string; updatedAt: Date; }`
- `index.ts` — barrel exporting services and types
