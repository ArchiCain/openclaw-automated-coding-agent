# Task: Update E2E Tests for Angular Material

## Goal
Update the Playwright E2E test suite to account for Angular Material's DOM differences versus MUI. Most tests use accessibility-first selectors and are already compatible ‚Äî only MUI-specific CSS class selectors need updating.

## Context
The E2E tests live at `projects/application/e2e/app/tests/`. The test suite covers:
- `auth/login.spec.ts` ‚Äî login flow
- `auth/protected-routes.spec.ts` ‚Äî access control
- `chat/send-message.spec.ts` ‚Äî chat messaging
- `user-management/create-user-login.spec.ts` ‚Äî user CRUD
- `hamburger-menu.spec.ts` ‚Äî navigation

The Angular rebuild preserves all accessibility selectors (role, label, placeholder, aria-label, form IDs). The only changes needed are MUI-specific CSS class selectors.

**Known selector changes** (from the plan):
| Current | Angular Material Equivalent |
|---------|---------------------------|
| `.MuiBackdrop-root` | `.cdk-overlay-backdrop` |

Search all test files for any other MUI-specific selectors (`.Mui*`) and update them.

## What to Build

Audit and update `projects/application/e2e/app/tests/`:

1. Search all `.spec.ts` files for `.Mui` class selectors
2. Replace each with the Angular Material / CDK equivalent:
   - `.MuiBackdrop-root` ‚Üí `.cdk-overlay-backdrop`
   - `.MuiDialog-root` ‚Üí `.mat-mdc-dialog-container` or `[role="dialog"]`
   - `.MuiCircularProgress-root` ‚Üí `mat-spinner` or `[role="progressbar"]`
   - Any other `.Mui*` ‚Üí use role/aria selectors where possible
3. Verify no React-specific selectors remain (e.g., `data-testid` values that were React-specific)
4. Ensure `playwright.config.ts` still targets `http://localhost:3000`

Do NOT change tests that already use role/label/placeholder selectors ‚Äî they work with Angular Material out of the box.

## Acceptance Criteria
- [x] No `.Mui*` CSS class selectors remain in any test file
- [x] All `.MuiBackdrop-root` replaced with `.cdk-overlay-backdrop`
- [x] Dialog selectors updated to CDK/Angular Material equivalents
- [x] Tests that use `getByRole`, `getByLabel`, `getByPlaceholder`, `getByText` are unchanged
- [x] `playwright.config.ts` unchanged (still targets localhost:3000)
- [x] Test file structure unchanged (same files, same test names)
