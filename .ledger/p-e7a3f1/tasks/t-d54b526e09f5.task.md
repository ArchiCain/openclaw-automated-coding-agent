# Task: Navigation Config Feature

## Goal
Build the `navigation-config` feature: TypeScript types for menu items and the menu structure definition. This is a pure data/config feature with no UI — it defines what navigation links exist and how they're structured, consumed by the navigation sidebar components.

## Context
The existing React implementation: `projects/application/frontend/app/src/features/navigation-config/`. It defines the menu structure (links to chat, smoke-tests, admin/users, etc.) and the `NavItem` type. The Angular version is a direct port — same structure, just typed for Angular/Angular Router.

## What to Build

Create `src/features/navigation-config/` with:

- `types.ts` — TypeScript interfaces:
  - `NavItem { label: string; icon: string; route?: string; children?: NavItem[]; requiredPermission?: string; }`
  - `NavSection { title?: string; items: NavItem[]; }`
- `navigation.config.ts` — the menu structure as a constant:
  - Main navigation: Chat (`/`), Smoke Tests (`/smoke-tests`)
  - Admin section: Users (`/admin/users`)
  - Use Material Icons names for `icon` field (e.g., `'chat'`, `'science'`, `'people'`)
  - Admin items should have `requiredPermission: 'admin'`
- `index.ts` — barrel: `export * from './types'; export { NAV_CONFIG } from './navigation.config';`

## Acceptance Criteria
- [ ] `NavItem` interface has `label`, `icon`, `route`, optional `children`, optional `requiredPermission`
- [ ] `NAV_CONFIG` exports the full menu structure matching existing routes
- [ ] Admin items are marked with `requiredPermission`
- [ ] No UI components (pure config/types)
- [ ] `ng build` succeeds
- [ ] Type-check passes
