# Task: Angular Project Scaffolding

## Goal
Replace the existing React/Vite app under `projects/application/frontend/app/` with a fresh Angular 19+ project scaffold. This establishes the build tooling, TypeScript configuration, and application entry point that all other frontend tasks depend on.

## Context
The current frontend is React 19 + Vite (`projects/application/frontend/app/`). The outer project structure (`dockerfiles/`, `chart/`, `Taskfile.yml`) is preserved — only the `app/` directory contents are replaced. The Angular app must maintain the `src/features/` layout for decomposition pipeline compatibility.

The outer `Taskfile.yml` at `projects/application/frontend/Taskfile.yml` references build commands — leave it as-is for now; a separate task (app-wiring) will update it.

## What to Build

**Delete** the existing React/Vite `app/` contents and create a fresh Angular CLI project in its place:

- `app/angular.json` — Angular workspace config; builder: `@angular-devkit/build-angular:application` (esbuild); project name: `frontend`; outputPath: `dist/frontend`; index: `src/index.html`; browser: `src/main.ts`; styles: `src/styles.scss`; assets: `src/assets`
- `app/package.json` — Angular 19+ dependencies: `@angular/core`, `@angular/common`, `@angular/forms`, `@angular/router`, `@angular/platform-browser`, `@angular/material`, `@angular/cdk`, `@angular/animations`, `socket.io-client`, `ngx-markdown`; devDependencies: `@angular/cli`, `@angular/compiler-cli`, `karma`, `karma-chrome-launcher`, `karma-jasmine`, `karma-jasmine-html-reporter`, `karma-coverage`, `jasmine-core`, `@types/jasmine`, `typescript`
- `app/tsconfig.json` — strict mode TypeScript base config
- `app/tsconfig.app.json` — extends base, includes `src/main.ts`
- `app/tsconfig.spec.json` — extends base, includes `src/**/*.spec.ts`
- `app/src/index.html` — Angular app shell with `<app-root>`
- `app/src/main.ts` — stub: `bootstrapApplication(AppComponent, appConfig)` (appConfig stub, full providers in app-wiring task)
- `app/src/app.component.ts` — stub root component: `selector: 'app-root'`, standalone, renders `<router-outlet>`
- `app/src/app.routes.ts` — empty routes array `[]` (routes filled in app-wiring task)
- `app/src/app.config.ts` — stub `appConfig` exporting `ApplicationConfig` with `provideRouter(routes)` only (interceptors/full providers in app-wiring task)
- `app/src/environments/environment.ts` — environment config with NO defaults; fields: `backendUrl: string`, `production: boolean`; values must be set at build time (no placeholder strings)
- `app/src/styles.scss` — global styles file importing Angular Material theme (to be filled in material-theme task)
- `app/src/assets/` — empty directory (`.gitkeep`)
- `app/src/features/` — empty directory (`.gitkeep`); all features added by subsequent tasks

## Acceptance Criteria
- [ ] `ng build` runs without error on a clean install (produces `dist/frontend/`)
- [ ] `ng test --watch=false` runs Karma/Jasmine test runner without config errors
- [ ] `src/features/` directory exists (pipeline compatibility)
- [ ] No Vite, Vitest, React, or MUI dependencies remain in `package.json`
- [ ] `environments/environment.ts` has no hardcoded default URL strings
- [ ] TypeScript strict mode is enabled in `tsconfig.json`
- [ ] Type-check passes (`ng build` or `tsc --noEmit`)
