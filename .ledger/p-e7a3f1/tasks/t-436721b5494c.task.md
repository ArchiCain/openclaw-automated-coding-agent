# Task: ThemeService and Theme Toggle Component

## Goal
Build the `theme` feature: an injectable `ThemeService` that manages dark/light mode state with backend persistence, and a `ThemeToggleComponent` that renders a toggle button. This replaces the React `useTheme` hook + `theme.api.ts` pattern with Angular-native services.

## Context
The existing React implementation lives at `projects/application/frontend/app/src/features/theme/`. It has `use-theme.tsx` (a React hook managing theme state), `theme.api.ts` (API calls to persist theme preference), and a toggle component. The Angular version converts these to a singleton `ThemeService` with `inject(HttpClient)` for API calls, and a standalone toggle component that injects the service.

The theme is toggled by adding/removing the `.dark-theme` CSS class on `document.body` (defined in the material-theme task SCSS). The backend endpoint for persistence is `PATCH /users/me/preferences` with body `{ theme: 'dark' | 'light' }` (same as current React app).
