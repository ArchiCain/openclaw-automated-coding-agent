# Task: WebSocket Service (Socket.IO + RxJS)

## Goal
Build the `api-client` feature: an injectable Angular service wrapping the Socket.IO client with RxJS Observables. This is the WebSocket layer used by the chat feature for real-time message streaming.

## Context
The existing React implementation at `projects/application/frontend/app/src/features/api-client/websocket-client.ts` wraps Socket.IO in a plain TypeScript class. The Angular version becomes a singleton `@Injectable` service that exposes RxJS Observables instead of callbacks.

The backend WebSocket namespaces are:
- `/mastra-chat` — send messages, receive streaming chunks
- `/mastra-chat-history` — conversation list operations

Events (same as current React app):
- Emit: `chat-message` with `{ message, agentId, threadId? }`
- Listen: `chat-chunk` (streaming token), `chat-complete`, `chat-error`
- Listen: `history-update` (conversation list changes)

Angular's `HttpClient` is the HTTP layer (provided via `provideHttpClient()` in `app.config.ts`). This feature only covers WebSocket.

## What to Build

Create `src/features/api-client/` with:

- `websocket.service.ts` — `@Injectable({ providedIn: 'root' })`:
  - Manages `socket: Socket` (from `socket.io-client`) per namespace
  - `connect(namespace: string): void` — establishes authenticated socket connection (with `withCredentials: true`)
  - `disconnect(namespace: string): void`
  - `emit(namespace: string, event: string, data: unknown): void`
  - `on<T>(namespace: string, event: string): Observable<T>` — returns cold Observable wrapping `socket.on()`
  - Handles reconnection; connects with `withCredentials: true` for cookie-based auth
- `websocket.types.ts` — TypeScript interfaces:
  - `ChatMessage`, `ChatChunk`, `ChatComplete`, `ChatError`, `HistoryUpdate`
- `index.ts` — barrel: `export { WebSocketService } from './websocket.service'; export * from './websocket.types';`

## Acceptance Criteria
- [ ] `WebSocketService` is `providedIn: 'root'` (no NgModule needed)
- [ ] `on<T>()` returns `Observable<T>` (not a callback)
- [ ] Socket connections use `withCredentials: true`
- [ ] `ng build` succeeds
- [ ] Type-check passes
