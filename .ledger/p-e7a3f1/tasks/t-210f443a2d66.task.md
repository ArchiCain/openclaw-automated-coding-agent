# Task: Auth Guards and HTTP Interceptors

## Goal
Build Angular Router guards and Angular HttpClient interceptors for the `keycloak-auth` feature. These replace React's `ProtectedRoute` component and Axios interceptors with Angular-native patterns: functional `canActivate` guards and functional `HttpInterceptorFn` interceptors.

## Context
The existing React auth protection uses a `ProtectedRoute` wrapper component (`keycloak-auth/components/protected-route.tsx`) and Axios interceptors in `api-client/api-client.ts`. Angular replaces both with:
- **Functional guards** on routes (`canActivate: [authGuard]`)
- **`HttpInterceptorFn` interceptors** registered in `app.config.ts` via `withInterceptors([])`

`AuthService` is built in the dependency task (t-5a23bd61342d). Guards and interceptors inject it.

## What to Build

Add to `src/features/keycloak-auth/` (extending the feature from the previous task):

- `guards/auth.guard.ts` — functional guard:
  ```typescript
  export const authGuard: CanActivateFn = () => {
    const auth = inject(AuthService);
    const router = inject(Router);
    return auth.isAuthenticated$.pipe(
      take(1),
      map(authenticated => authenticated || router.createUrlTree(['/login']))
    );
  };
  ```
- `guards/permission.guard.ts` — functional guard factory:
  - `permissionGuard(permission: string): CanActivateFn` — checks `AuthService.hasPermission(permission)`; redirects to `/` if denied
- `interceptors/auth.interceptor.ts` — `HttpInterceptorFn`:
  - Adds `withCredentials: true` to every request (so HTTP-only cookies are sent)
- `interceptors/token-refresh.interceptor.ts` — `HttpInterceptorFn`:
  - On 401 response: call `AuthService.refresh()`, if successful retry the original request once; if refresh fails, call `AuthService.logout()` and redirect to `/login`
  - Queue concurrent requests during an in-progress refresh (use a `BehaviorSubject` as a lock)
- `interceptors/activity.interceptor.ts` — `HttpInterceptorFn`:
  - On each outbound request, notify `AuthService` of activity to reset inactivity timer
  - `AuthService` must expose a `recordActivity(): void` method (add it in this task or coordinate with auth-service task)

Update `index.ts` to also export:
- `authGuard`, `permissionGuard`
- `authInterceptor`, `tokenRefreshInterceptor`, `activityInterceptor`

## Acceptance Criteria
- [ ] `authGuard` redirects unauthenticated users to `/login`
- [ ] `permissionGuard` redirects insufficient-role users to `/`
- [ ] `authInterceptor` sets `withCredentials: true` on all requests
- [ ] `tokenRefreshInterceptor` retries once after a successful 401 refresh
- [ ] `activityInterceptor` calls `AuthService.recordActivity()` per request
- [ ] All interceptors are `HttpInterceptorFn` (not class-based)
- [ ] All guards are functional (`CanActivateFn`)
- [ ] `ng build` succeeds
- [ ] Type-check passes
