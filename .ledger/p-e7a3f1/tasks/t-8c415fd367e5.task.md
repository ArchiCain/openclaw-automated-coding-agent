# Task: User Management UI Components and Pages

## Goal
Build all user management UI components and routed pages for the `user-management` feature: users table with search/sort/pagination, user create/edit form, and delete confirmation. These are the admin pages at `/admin/users`, `/admin/users/new`, and `/admin/users/:id`.

## Context
The existing React implementation: `projects/application/frontend/app/src/features/user-management/pages/` and `components/`. The Angular version uses Angular Material `mat-table` with `MatPaginator` and `MatSort`, `mat-form-field` for the search input and user form fields.

`UserManagementService` is built in the dependency task (t-77f36a6d415c). `ConfirmationDialogComponent` (task t-c06bdcef4e32) is used for delete confirmation. The `permissionGuard` (task t-210f443a2d66) protects these routes.

**Important for E2E compatibility**: The user form uses `id="email"` for the email field (current E2E test: `input[id="email"]`).

## What to Build

Extend `src/features/user-management/` with:

- `components/users-table/users-table.component.ts` — standalone:
  - Imports: `[MatTableModule, MatPaginatorModule, MatSortModule, MatFormFieldModule, MatInputModule, MatButtonModule, MatIconModule, MatProgressSpinnerModule, CommonModule, RouterModule]`
  - Injects `UserManagementService` and `MatDialog`
  - `dataSource = new MatTableDataSource<User>()`
  - Columns: username, email, roles, actions (edit/delete)
  - Search input with debounce calls `getUsers({ search })`
  - `MatPaginator` and `MatSort` wired to `dataSource`
  - Delete button opens `ConfirmationDialogComponent`, then calls `deleteUser()`
- `components/users-table/users-table.component.html` — template
- `components/user-form/user-form.component.ts` — standalone, reused for both create and edit:
  - Form fields: username, email (id="email"), firstName, lastName, password (create only), roles (checkboxes)
  - `@Input() userId?: string` — if set, loads user and switches to edit mode
  - `@Output() saved = new EventEmitter<User>()`
  - On submit: calls `createUser()` or `updateUser()` accordingly
- `components/user-form/user-form.component.html` — template
- `pages/users-list.page.ts` — routed standalone component for `/admin/users`:
  - Renders `UsersTableComponent`
  - "Add User" button navigates to `/admin/users/new`
- `pages/create-user.page.ts` — routed standalone component for `/admin/users/new`:
  - Renders `UserFormComponent`
  - On `saved` navigates to `/admin/users`
- `pages/edit-user.page.ts` — routed standalone component for `/admin/users/:id`:
  - Gets `:id` from `ActivatedRoute`
  - Renders `UserFormComponent` with `[userId]="id"`
  - On `saved` navigates to `/admin/users`

Update `index.ts` to export all components and pages.
