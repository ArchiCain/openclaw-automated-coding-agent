# Task: Testing Tools Feature (Smoke Tests Page)

## Goal
Build the `testing-tools` feature: the smoke tests page at `/smoke-tests` with a backend health check and database client UI. Direct port of the React implementation to standalone Angular components.

## Context
The existing React implementation: `projects/application/frontend/app/src/features/testing-tools/`. It renders a page with health check results (calls backend health endpoint) and a basic DB client UI for running queries. This is an internal developer tool, not a production feature.

Backend endpoints:
- `GET /health` — returns health status
- `POST /db/query` — body: `{ query: string }` → query results (if such endpoint exists; port as-is from React)

## What to Build

Create `src/features/testing-tools/` with:

- `components/health-check/health-check.component.ts` — standalone:
  - Injects `HttpClient`
  - `checkHealth(): void` — calls `GET /health`, displays results
  - Shows loading state, success/error status
  - `<button mat-raised-button>Run Health Check</button>`
- `components/db-client/db-client.component.ts` — standalone (if present in React version):
  - Text area for SQL/query input
  - Run button + results display
- `pages/smoke-tests.page.ts` — routed standalone component for `/smoke-tests`:
  - Composes health check and DB client components
  - Page title: "Smoke Tests"

- `index.ts` — barrel exporting the page component

## Acceptance Criteria
- [ ] `SmokeTestsPage` renders at `/smoke-tests`
- [ ] Health check button calls `GET /health` and displays response
- [ ] All components are standalone
- [ ] `ng build` succeeds
- [ ] Type-check passes
