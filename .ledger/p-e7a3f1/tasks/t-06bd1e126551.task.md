# Task: Login Page Component

## Goal
Build the login page component for the `keycloak-auth` feature. This is the standalone Angular component rendered at the `/login` route, replacing the React `login.tsx` page with Angular Material form components.

## Context
The existing React login page: `projects/application/frontend/app/src/features/keycloak-auth/components/login.tsx` and `login-form.tsx`. It renders a centered card with username/password fields and a sign-in button. After successful login it redirects to `/`. On failed login it shows an error message.

The `AuthService` (from task t-5a23bd61342d) provides the `login()` method. This component injects it directly.

**Important for E2E compatibility**: Preserve these accessibility selectors from the current app (E2E tests use them):
- Heading: `getByRole('heading', { name: /sign in/i })`
- Username: `getByLabel(/username/i)` with `id="username"`
- Password: `getByLabel(/password/i)` with `id="password"`
- Submit button: `getByRole('button', { name: /sign in/i })`

## What to Build

Add to `src/features/keycloak-auth/` (extending the feature):

- `components/login/login.component.ts` — standalone `@Component`:
  - `selector: 'app-login'`
  - Imports: `[ReactiveFormsModule, MatCardModule, MatFormFieldModule, MatInputModule, MatButtonModule, MatProgressSpinnerModule, CommonModule]`
  - `loginForm = new FormGroup({ username: new FormControl('', Validators.required), password: new FormControl('', Validators.required) })`
  - `onSubmit()` — calls `authService.login()`, on success navigates to `/`, on error shows error message
  - Loading state during submit
- `components/login/login.component.html` — template:
  - Centered layout (full-height flex column, center align)
  - `<mat-card>` with app name/logo
  - `<h1>` or `<mat-card-title>` with "Sign In" text (for `getByRole('heading', { name: /sign in/i })`)
  - `<mat-form-field>` with `<mat-label>Username</mat-label>` and `<input matInput id="username" formControlName="username">`
  - `<mat-form-field>` with `<mat-label>Password</mat-label>` and `<input matInput type="password" id="password" formControlName="password">`
  - `<button mat-raised-button color="primary">Sign In</button>`
  - Error message display for failed login

Update `index.ts` to also export `LoginComponent`.

## Acceptance Criteria
- [ ] `LoginComponent` is standalone (no NgModule)
- [ ] Form uses Angular Reactive Forms with validators
- [ ] Username input has `id="username"`, password has `id="password"`
- [ ] "Sign In" text is in a heading role element
- [ ] Successful login navigates to `/`
- [ ] Failed login displays error message
- [ ] Loading state prevents double-submit
- [ ] `ng build` succeeds
- [ ] Type-check passes
