# Task: FastAPI Backend — Project Scaffold, Config, and Database Layer

## Goal
Create the `projects/application/backend-python/` project from scratch. This is the foundational scaffold for a new FastAPI (Python 3.12) backend that replicates all NestJS endpoints. Establish the project layout, configuration, async database layer, and project tooling so that all subsequent feature tasks can build on top of it.

## Context
This is a brand-new Python project at `projects/application/backend-python/`. The existing NestJS backend at `projects/application/backend/` must NOT be modified. This Python backend connects to the same PostgreSQL database (same tables/schema) and the same Keycloak realm. The plan file at `.ledger/p-f4a512/plan.md` contains full architecture details.

The project structure mirrors the NestJS backend's feature-based pattern but uses Python idioms:
```
projects/application/backend-python/
├── app/
│   ├── main.py                    # FastAPI app factory, middleware, lifespan
│   ├── config.py                  # Settings from env vars (pydantic-settings)
│   └── features/
│       └── database/              # SQLAlchemy async engine, base model, session
├── alembic/                       # DB migrations config (reuse existing schema)
├── requirements.txt
├── requirements-dev.txt
└── Taskfile.yml
```

App wiring (including all routers in main.py) is handled by a separate task. This task delivers only the scaffold and database layer.

## What to Build

### `app/config.py`
Pydantic-settings `Settings` class reading these env vars:
- `PORT` (default 8080)
- `DATABASE_URL` (PostgreSQL async URL, e.g. `postgresql+asyncpg://...`)
- `KEYCLOAK_URL`, `KEYCLOAK_REALM`, `KEYCLOAK_CLIENT_ID`, `KEYCLOAK_CLIENT_SECRET`
- `ALLOWED_ORIGINS` (comma-separated string)
- `ANTHROPIC_API_KEY`, `OPENAI_API_KEY`

### `app/main.py`
FastAPI app factory with:
- CORSMiddleware using `ALLOWED_ORIGINS` from config
- Lifespan context manager (startup/shutdown hooks for DB engine, etc.)
- Global exception handlers
- A placeholder router include point (comment: `# Feature routers registered here`)
- No routes yet — those come from feature tasks

### `app/features/database/`
Async SQLAlchemy 2.0 setup:
- `engine.py` — async engine from `DATABASE_URL`, `async_sessionmaker`
- `base.py` — declarative `Base` for all models
- `session.py` — FastAPI dependency `get_db_session()` yielding `AsyncSession`
- `__init__.py` — re-exports

### `alembic/`
Alembic configuration pointing to `DATABASE_URL`. Since schema already exists (created by NestJS migrations), configure Alembic to treat existing tables as present. Include `env.py` pointing at the SQLAlchemy `Base`.
