# Task: FastAPI Backend — Docker Build Files

## Goal
Create the Docker build configuration for the `backend-python` project: a production multi-stage Dockerfile and a local development Dockerfile with hot reload.

## Context
The new FastAPI backend lives at `projects/application/backend-python/`. Mirror the NestJS backend Docker patterns from `projects/application/backend/dockerfiles/` but adapted for Python/uvicorn.

Both Dockerfiles are in `projects/application/backend-python/dockerfiles/`.

## What to Build

### `projects/application/backend-python/dockerfiles/prod.Dockerfile`
Multi-stage build:
1. **Stage 1 (builder):** `python:3.12-slim` — install dependencies from `requirements.txt` into a virtualenv
2. **Stage 2 (runtime):** `python:3.12-slim` — copy virtualenv, copy `app/` source, expose 8080, start uvicorn

```dockerfile
# Stage 1: Dependencies
FROM python:3.12-slim AS builder
WORKDIR /app
COPY app/requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Stage 2: Production runtime
FROM python:3.12-slim
WORKDIR /app
COPY --from=builder /usr/local/lib/python3.12 /usr/local/lib/python3.12
COPY --from=builder /usr/local/bin/uvicorn /usr/local/bin/uvicorn
COPY app/ .
EXPOSE 8080
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8080"]
```
(Adapt virtualenv copy path as needed for your pip install approach.)

### `projects/application/backend-python/dockerfiles/local.Dockerfile`
Development image with hot reload:
- `python:3.12-slim` base
- Install `requirements.txt` AND `requirements-dev.txt`
- Mount source as volume (volume defined in compose.yml by infra task)
- Start uvicorn with `--reload`
```dockerfile
FROM python:3.12-slim
WORKDIR /app
COPY app/requirements.txt app/requirements-dev.txt ./
RUN pip install --no-cache-dir -r requirements.txt -r requirements-dev.txt
COPY app/ .
EXPOSE 8080
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8080", "--reload"]
```
