# Task: FastAPI Backend ‚Äî Mastra Agents and Socket.IO Chat

## Goal
Implement the AI agent integration and WebSocket chat feature for the FastAPI backend. This replicates the NestJS `/mastra-chat` and `/mastra-chat-history` Socket.IO gateways, and provides AI agent endpoints using Anthropic (claude-sonnet) and OpenAI (gpt-4o) models.

## Context
This task builds on the project scaffold from task `t-3f8b41a454b7`. The NestJS mastra-agents feature at `projects/application/backend/app/src/features/mastra-agents/` is the reference implementation.

**WebSocket namespaces** (Socket.IO, same as NestJS):
- `/mastra-chat` ‚Äî real-time streaming chat with AI agents
- `/mastra-chat-history` ‚Äî chat history retrieval

The NestJS backend uses the Mastra framework. The Python backend should replicate the same API surface using the Anthropic Python SDK and OpenAI Python SDK directly (no Mastra in Python). The streaming response format and Socket.IO event names must match so the Angular frontend works with both backends without changes.

**AI models used (same as NestJS):**
- Anthropic: `claude-sonnet-4-5` (or equivalent current claude-sonnet)
- OpenAI: `gpt-4o`

Socket.IO is provided by `python-socketio` with ASGI mode, mounted into the FastAPI app.

## What to Build

### `app/features/mastra_agents/`
- `models.py` ‚Äî Pydantic models: `ChatMessage`, `ChatRequest`, `AgentInfo`, `ChatHistoryEntry`
- `agent_service.py` ‚Äî async service that:
  - Lists available agents (returns name, description, model for each)
  - Streams chat completions from Anthropic/OpenAI based on selected agent
  - Uses `ANTHROPIC_API_KEY` and `OPENAI_API_KEY` from config
- `socket_handlers.py` ‚Äî python-socketio event handlers for:
  - `/mastra-chat` namespace: `message` event ‚Üí stream AI response tokens back via `token` events, `complete` event when done, `error` on failure
  - `/mastra-chat-history` namespace: `get_history` event ‚Üí returns conversation history
- `router.py` ‚Äî REST endpoints:
  - `GET /agents` ‚Äî list available agents (name, id, model info)
  - `POST /agents/:id/run` ‚Äî execute an agent with a prompt, return full response
- `__init__.py`

### Update `app/main.py`
Mount python-socketio ASGI app alongside FastAPI:
```python
import socketio
sio = socketio.AsyncServer(async_mode='asgi', cors_allowed_origins=settings.allowed_origins_list)
socket_app = socketio.ASGIApp(sio, app)
# The combined socket_app is what uvicorn serves
```
Register the mastra_agents router.

### Socket.IO event contract (must match NestJS exactly)
**`/mastra-chat` namespace:**
- Client emits: `{ event: 'message', data: { agentId: string, message: string, conversationId?: string } }`
- Server emits: `{ event: 'token', data: { token: string } }` (streaming)
- Server emits: `{ event: 'complete', data: { conversationId: string } }` (done)
- Server emits: `{ event: 'error', data: { message: string } }` (on error)

**`/mastra-chat-history` namespace:**
- Client emits: `{ event: 'get_history', data: { conversationId: string } }`
- Server emits: `{ event: 'history', data: ChatHistoryEntry[] }`

Read the actual NestJS gateway implementations at `projects/application/backend/app/src/features/mastra-agents/gateways/` to confirm exact event names before implementing.
