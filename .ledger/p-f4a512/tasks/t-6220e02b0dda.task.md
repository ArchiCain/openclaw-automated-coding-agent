# Task: Frontend — Backend Selector HTTP Interceptor and WebSocket URL Wiring

## Goal
Add an Angular HTTP interceptor that prepends the FastAPI backend base URL to all API requests when FastAPI is selected, and update the WebSocket service to use the selected backend URL for Socket.IO connections.

## Context
`BackendConfigService` (created in task `t-575d106b8c34`) provides `getBaseUrl()` returning `''` for NestJS or `'http://api-python.mac-mini'` for FastAPI.

The Angular app currently uses three functional interceptors registered in `app.config.ts`:
- `authInterceptor` — adds `withCredentials: true`
- `tokenRefreshInterceptor` — handles 401 and token refresh
- `activityInterceptor` — tracks user activity

A new `backendUrlInterceptor` must be added that prepends the base URL for non-absolute API requests. It should NOT rewrite requests that already have an absolute URL (like direct Keycloak calls if any) or requests to `/login`.

The `WebSocketService` at `projects/application/frontend/app/src/features/api-client/websocket.service.ts` currently uses relative namespace paths for Socket.IO. It should use the full backend URL when FastAPI is selected.

## What to Build

### `app/src/features/backend-selector/backend-url.interceptor.ts`
Functional HTTP interceptor:
```typescript
export const backendUrlInterceptor: HttpInterceptorFn = (req, next) => {
  const backendConfig = inject(BackendConfigService);
  const baseUrl = backendConfig.getBaseUrl();
  if (baseUrl && !req.url.startsWith('http')) {
    return next(req.clone({ url: `${baseUrl}${req.url}` }));
  }
  return next(req);
};
```

### Update `app/src/features/api-client/websocket.service.ts`
Inject `BackendConfigService` and prepend base URL to Socket.IO namespace:
```typescript
connect(namespace: string): void {
  const baseUrl = this.backendConfig.getBaseUrl();
  const socketUrl = baseUrl ? `${baseUrl}${namespace}` : namespace;
  // use socketUrl when creating io() connection
}
```

### Update `app/src/app.config.ts`
Add `backendUrlInterceptor` to the interceptors list in `provideHttpClient`:
```typescript
provideHttpClient(
  withInterceptors([backendUrlInterceptor, authInterceptor, tokenRefreshInterceptor, activityInterceptor])
)
```
Place `backendUrlInterceptor` first so the URL is set before auth headers are applied.

### Update `app/src/features/backend-selector/index.ts`
Export `backendUrlInterceptor`.
