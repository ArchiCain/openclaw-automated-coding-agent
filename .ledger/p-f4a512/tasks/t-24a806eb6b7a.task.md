# Task: FastAPI Backend — Health, Auth, Users, and Theme Features

## Goal
Implement the four core API features in the FastAPI backend: health check, Keycloak cookie-based auth, user management (CRUD via Keycloak admin), and user theme preferences. These replicate the equivalent NestJS endpoints exactly.

## Context
This task builds on top of the project scaffold created in task `t-3f8b41a454b7` (which provides `app/main.py`, `app/config.py`, `app/features/database/`). Wire all routers into `main.py` here.

The NestJS backend at `projects/application/backend/app/src/features/` implements the same logic — use it as the reference for endpoint shapes, cookie handling, and Keycloak integration patterns.

**Keycloak authentication flow** (same as NestJS):
- JWT tokens in HTTP-only cookies (`access_token`, `refresh_token`)
- Global auth dependency — all routes protected by default unless marked public
- Login exchanges username/password with Keycloak token endpoint, sets cookies
- Logout calls Keycloak logout endpoint, clears cookies
- Check (`POST /auth/check`) verifies current cookie-based JWT, returns user profile
- Refresh (`POST /auth/refresh`) uses refresh token cookie to get new access token

**User management** calls Keycloak Admin REST API (not the app database directly for user data — users live in Keycloak). Theme preferences are stored in PostgreSQL in the existing `user_theme` table (or equivalent).

## What to Build

### `app/features/health/`
- `router.py` — `GET /health` returning `{"status": "ok", "timestamp": <ISO>, "service": "backend-python"}`
- `__init__.py`

### `app/features/auth/`
- `models.py` — Pydantic request/response models (LoginRequest, TokenResponse, UserProfile)
- `keycloak_client.py` — async httpx client wrapping Keycloak token/logout/userinfo/refresh endpoints
- `dependencies.py` — `get_current_user` FastAPI dependency extracting and validating JWT from cookie
- `router.py` — login, logout, check, refresh endpoints; sets/clears HTTP-only cookies
- `__init__.py`

### `app/features/user_management/`
- `models.py` — Pydantic models (UserResponse, CreateUserRequest, UpdateUserRequest)
- `keycloak_admin.py` — async httpx client for Keycloak Admin REST API (list, create, get, update, delete users)
- `router.py` — all `/users` and `/users/:id` endpoints; `/users/me` returns current user from JWT
- `__init__.py`

### `app/features/theme/`
- `models.py` — ThemePreference (light/dark), ThemeResponse, UpdateThemeRequest
- `service.py` — async service reading/writing `user_theme` table via SQLAlchemy. Look up the existing table schema from NestJS's entity at `projects/application/backend/app/src/features/theme/` or `features/typeorm-database-client/entities/`
- `router.py` — `GET /users/me/preferences`, `PATCH /users/me/preferences`
- `__init__.py`

### Update `app/main.py`
Register all feature routers:
```python
app.include_router(health_router)
app.include_router(auth_router, prefix="/auth")
app.include_router(users_router, prefix="/users")
app.include_router(theme_router)
```

## Acceptance Criteria
- [ ] `GET /health` returns `{"status": "ok"}` with HTTP 200, no auth required
- [ ] `POST /auth/login` authenticates with Keycloak, sets `access_token` and `refresh_token` HTTP-only cookies
- [ ] `POST /auth/logout` clears cookies and calls Keycloak logout
- [ ] `POST /auth/check` returns user profile when valid JWT cookie present; 401 otherwise
- [ ] `POST /auth/refresh` exchanges refresh token for new access token
- [ ] `GET /users` returns list of users from Keycloak admin API
- [ ] `POST /users` creates a user in Keycloak admin API
- [ ] `GET /users/me` returns current authenticated user's profile
- [ ] `GET /users/:id`, `PATCH /users/:id`, `DELETE /users/:id` operate via Keycloak admin API
- [ ] `GET /users/me/preferences` reads theme from database
- [ ] `PATCH /users/me/preferences` writes theme to database
- [ ] All protected routes return HTTP 401 when called without valid auth cookie
- [ ] No import errors; all dependencies resolve
- [ ] Type annotations on all functions and models
