# Task: E2E Tests — Backend Selector Toggle

## Goal
Add Playwright E2E tests verifying the backend selector toggle appears in the app header and that switching backends persists the selection and changes API call URLs.

## Context
E2E tests are at `projects/application/e2e/app/tests/`. Existing test suites cover auth, chat, and user management. Add a new `backend-selector/` directory.

The backend selector toggle appears in `AppHeaderComponent`. Tests should:
1. Verify the toggle exists and defaults to "NestJS"
2. Switch to "FastAPI" and verify the selection persists in localStorage
3. Verify after page reload the "FastAPI" selection is retained

Note: These tests run against the live deployed app where the FastAPI backend may or may not be fully operational. Tests should focus on UI state (toggle visible, selection persists), not on actual API functionality (that's covered by the Python backend's pytest suite).

## What to Build

### `projects/application/e2e/app/tests/backend-selector/`
**`backend-selector.spec.ts`:**
```typescript
test.describe('Backend Selector', () => {
  test('shows toggle in header', ...)       // toggle visible, default=NestJS
  test('switches to FastAPI and persists', ...) // click FastAPI, check localStorage
  test('retains selection after reload', ...) // reload page, selection still FastAPI
  test('switches back to NestJS', ...)     // click NestJS, check localStorage
})
```

Tests should log in (reuse auth helpers from `tests/auth/` if available) before accessing the header.

## Acceptance Criteria
- [ ] `tests/backend-selector/backend-selector.spec.ts` created
- [ ] Test: header contains backend selector toggle
- [ ] Test: default selection is "NestJS"
- [ ] Test: switching to FastAPI stores `preferred-backend=fastapi` in localStorage
- [ ] Test: page reload retains FastAPI selection
- [ ] Tests pass against the deployed frontend (where toggle is present)
- [ ] Tests are isolated (clean up localStorage after each test)
